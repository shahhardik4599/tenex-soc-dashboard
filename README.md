# TENEX SOC Analyst Dashboard

An enterprise-grade Security Operations Center (SOC) dashboard that ingests raw server access logs (Apache/Nginx), detects malicious activity using a multi-layered Machine Learning approach, and enriches threats with GenAI-powered tactical summaries.


## 🧠 AI & Threat Detection Architecture

The backend pipeline processes logs through a three-layer detection architecture designed for high accuracy and low latency:

### 1. Layer 1: Deterministic Heuristics
* **Threat Intelligence:** Matches incoming IP addresses against a local STIX JSON feed of known malicious actors.
* **Velocity Rules (Brute Force):** Identifies attacks by flagging IPs exceeding specific request-per-minute thresholds.
* **Sensitive Probing:** Detects unauthorized attempts to access administrative or sensitive endpoints (e.g., `/.env`, `/admin`, `/.git`).

### 2. Layer 2: Machine Learning (Behavioral Anomaly)
Traffic that passes Layer 1 is analyzed using an **Isolation Forest** model (`scikit-learn`). 
* **Approach:** By analyzing `status_code` and `response_size`, the model identifies zero-day exploits or stealthy data exfiltration that doesn't match known signatures.

### 3. Layer 3: AI Analyst Enrichment (Groq + Llama 3.1)
Anomalies are compressed into unique "attack signatures" to avoid redundant API calls. These are sent to the **Groq API (Llama-3.1-8b-instant)**, which acts as a Senior SOC Analyst to provide a 1-sentence tactical explanation for the security team.

## 🚀 Quick Start (Dockerized)

The easiest way to run the entire stack (Database, Backend, and Frontend) is using Docker Compose.

### Prerequisites
* [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.
* A [Groq API Key](https://console.groq.com/).

### Setup Instructions

1.  **Clone the Repository:**
    ```bash
    git clone <your-repo-url>
    cd tenex-soc-dashboard
    ```

2.  **Configure Environment Variables:**
    Create a `.env` file inside the `backend/` folder:
    ```env
    GROQ_API_KEY=your_groq_api_key_here
    ```

3.  **Launch the Application:**
    In the root directory (where `docker-compose.yml` is located), run:
    ```bash
    docker-compose up --build
    ```
    *Note: The first build may take a few minutes as it compiles the Next.js frontend and installs Python dependencies.*

4.  **Access the Dashboard:**
    * **Frontend Dashboard:** [http://localhost:3000](http://localhost:3000)
    * **Backend API Docs:** [http://localhost:8000/docs](http://localhost:8000/docs)

## 🧪 Usage & Testing

Once the Docker containers are running, follow these steps to see the detection engine in action:

1. **Create an Account:** Navigate to [http://localhost:3000](http://localhost:3000), click on **Sign Up**, and create a new local account to manage your SOC sessions.
2. **Upload Sample Data:** Locate the included test file at `data/sample_access.log` (or download it directly from the repository).
3. **Run the Detection Pipeline:** Click the **Upload Access Log** button on the dashboard and select the `sample_access.log` file.
4. **View the Results:** Watch the Live Threat Timeline populate instantly, identifying Threat Intel hits, Brute Force attacks, and ML anomalies, all enriched with GenAI explanations.

## 📖 Key Features

* **Live Threat Timeline:** Interactive Recharts visualization showing traffic spikes and anomaly concentrations.
* **Automated Enrichment:** Every detected threat includes a reason generated by Llama 3.1.
* **Persistent History:** Historical batches are stored in PostgreSQL and can be reloaded via the Sidebar.
* **Intelligent Filtering:** Toggle between "All Traffic" (live only) and specific threat categories.
